# 环境管理设计：统一使用 uv

> 日期：2026-07-05
> 状态：已批准，待实施
> 范围：全项目（mission1 / mission2 / mission3）

## 1. 背景与问题

项目已「半 uv 化」，但存在严重的依赖声明不一致（技术债）：

| 文件 | 依赖声明 | 问题 |
|---|---|---|
| `requirements.txt` | numpy**<2.0**, scipy, matplotlib, pytest, streamlit | numpy 上限 **<2.0**；缺 pillow |
| `pyproject.toml` | numpy**>=2.5.1**, matplotlib, pillow | numpy 下限 **>=2.5.1**（与 requirements 直接冲突）；缺 scipy/pytest/streamlit |
| `uv.lock` | 已锁定（基于 pyproject） | 与 requirements.txt 不一致 |
| `.venv/` | Python 3.12.13，uv 创建 | OK |
| `CLAUDE.md` | 仍称「根目录已有统一 requirements.txt」 | 未提 uv，与现状不符 |

`requirements.txt` 的 `numpy<2.0` 与 `pyproject.toml` 的 `numpy>=2.5.1` **直接矛盾**。本设计的目标是消除双轨，让 uv 成为唯一真相源。

实际依赖使用（grep 验证）：

- 任务一（`mission1/heat.py`）：numpy、scipy、matplotlib、**pillow**（`PillowWriter` 存 GIF 动画）
- 任务二（`mission2/three_switch.py`）：numpy、matplotlib
- 任务三（`mission3/`）：numpy、scipy（`scipy.stats`）、matplotlib、pytest、streamlit

## 2. 已批准决策

1. **删 `requirements.txt`**，`pyproject.toml` + `uv.lock` 为唯一真相源。
2. **扁平一套主依赖**（不做 extras 分组）：numpy / scipy / matplotlib / pillow / pytest / streamlit 全部作主依赖。
3. **全链路同步**：同步修改 `CLAUDE.md`、各 mission README、复现说明，确保文档与代码一致。

## 3. 整体架构

```
pyproject.toml      ← 人写，唯一手动维护的依赖声明
      │ uv lock
   uv.lock          ← 机器维护，锁全传递依赖，提交 git
      │ uv sync
   .venv/           ← 本地，gitignore，Python 3.12.13

.python-version     ← 内容 "3.12"，uv 自动据此选/建 Python
```

信息流单向：`pyproject.toml → uv.lock → .venv`。

## 5. 依赖版本处理

以现有 `uv.lock` 为准，**不主动升级**，只修矛盾：

```toml
[project]
name = "math-thinking-practice"
version = "0.1.0"
requires-python = ">=3.12"
dependencies = [
    "matplotlib>=3.11.0",
    "numpy>=2.5.1",
    "pillow>=12.3.0",
    "scipy>=1.10",
    "pytest>=7.0",
    "streamlit>=1.30",
]
```

要点：
- `numpy` 去掉 `<2.0` 错误上限，下限 `>=2.5.1` 跟随现有 lock。
- 新增 `scipy>=1.10`、`pytest>=7.0`、`streamlit>=1.30`。
- 保留现有 `matplotlib>=3.11.0`、`pillow>=12.3.0`。
- 保留 `requires-python = ">=3.12"`（system Python 3.9.6 不够，必须由 uv 准备 3.12）。

## 6. 运行与复现命令

| 场景 | 命令 |
|---|---|
| 首次 / 日常 | `uv sync` |
| 跑脚本 | `uv run python mission1/heat.py` |
| 跑测试 | `uv run python -m pytest mission1/test_heat.py` |
| 仪表盘 | `uv run streamlit run mission3/app.py` |
| 任务三全量 | `uv run python mission3/run_all.py` |
| 他机 / 打包复现 | `uv sync --frozen` |

`uv run` 自动确保 venv 同步后再跑，**无需手动 activate**。

## 7. 提交打包说明（四目录归档）

按 CLAUDE.md 的提交打包要求，环境复现归入「源代码与数据」目录：

- 提交 `pyproject.toml`、`uv.lock`、`.python-version`。
- 不提交 `.venv/`（已被 gitignore）。
- 评阅人/组员复现：`uv sync --frozen` 后 `uv run ...`。
- `requirements.txt` 不再作为提交物。
