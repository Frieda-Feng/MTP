| 2026-07-05 | 【任务二】验证三控开关逻辑时，是否需要先约定初态 L(0,0,0) 的值？如果约定 L(0,0,0)=1（初始灯亮），会不会得到不同的逻辑表达式？ | 手工演算发现：若约定 L(0,0,0)=1，则真值表整体取反，得到 $L=\overline{S1\oplus S2\oplus S3}$，等价于 XNOR。两种约定都满足"任一开关翻转→输出变化"，区别仅在意愿（灯亮是"正常工作"还是"故障报警"）。从工程角度看，选 L(0,0,0)=0（全关时灯灭）更符合日常直觉 | 初态约定不影响逻辑正确性，只决定 L 是 XOR 还是 XNOR。选 XOR、初态灯灭，是工程直觉而非数学推导的要求。报告中的证明（§3.5）适用于任意初态约定 |

# 七、总结与反思

本次实践围绕三个数学分支——线性代数（任务一）、离散数学（任务二）、概率统计（任务三）——各自由一名成员独立深度执行，遵循统一的"数学模型层 → 计算实现层 → 可视化验证层"三层结构完成。主要收获如下：

（1）**数学建模能力**：三个任务均展示了"把实际问题翻译成数学语言"的完整过程。任务一把铁板稳态热传导翻译成线性方程组 Ax=b，沿铁棍（1D）→ 铁板 2×2 → 铁板 30×30 三级递进，逐步揭示迭代法的登场动机；任务二把三控开关翻译成布尔逻辑系统，用真值表枚举 8 种状态，推导出 L=S1⊕S2⊕S3 这一最简表达式；任务三把空调故障间隔的估计问题翻译成 Bootstrap 重采样框架，用经验分布近似中位数的抽样分布。三个任务数学分支不同，但核心思路一致：定义变量 → 建立关系 → 推导/求解 → 验证。

（2）**数学原理的深度理解**：实践过程中遇到若干"直觉以为有问题，推导后才发现是数学性质"的时刻——这些时刻恰是本课程最深刻的学习收获。任务一发现"步差 10⁻⁶ ≠ 真误差 10⁻⁴"，推出 tol/(1−ρ) 上界后才理解谱半径 ρ 接近 1 时的放大效应；任务三发现 BCa 与 percentile 几乎重合，推导后确认是中位数（非光滑泛函）在偶数 n 下加速度恒为零的固有性质；覆盖率 0.934 低于名义 0.95，查文献确认是 percentile 一阶精度在偏态小样本下的已知特征。这些经历教会我们：**遇到异常结果，先怀疑代码、再怀疑方法，最后可能是方法本身的性质——推导一遍才敢下结论。**

（3）**编程与工程能力**：三个任务均使用 Python + numpy + matplotlib，依赖管理统一为 uv。技术栈覆盖面较广——任务一涉及数值线性代数（稀疏矩阵组装、迭代法实现、谱半径计算），任务二涉及布尔逻辑编程与自动验证（itertools 枚举、24 种翻转穷举），任务三涉及向量化统计模拟（numpy 批量重采样、percentile/BCa 置信区间、覆盖率实验）。过程中解决了多个工程问题：matplotlib 中文 CJK 字体、Windows GBK 编码不兼容 emoji、Python 模块导入路径、重复调用导致的输出污染等。统一使用 uv 管理依赖，从仓库根或各任务目录内均可一键运行。

（4）**可视化表达能力**：可视化不是装饰，而是结论的支撑。三个任务共产出 15 张静态图 + 3 张 GIF 动画，每张图配有"展示了什么 / 观察到什么 / 支持什么结论"的三段式说明。任务一用热力图展示温度场、半对数误差曲线展示收敛速度、GIF 动画展示迭代逐步逼近过程；任务二用 3D 状态立方体展示拓扑结构、颜色矩阵展示真值表、验证矩阵给出量化结论；任务三用直方图+KDE 展示数据分布、演化图展示 Bootstrap 分布如何随 B 增大而稳定、三因素对比图展示样本量/重采样次数/离群点的影响。图标架构遵循"结构→过程→结果"三层叙事。

（5）**交互式展示与加分项**：任务二额外编写了自包含 HTML 交互页面（双击浏览器打开），实现开关点击切换、灯发光动画、公式逐步展开和真值表实时高亮，让数学模型变成可触摸的实物。任务三额外编写了 Streamlit 交互仪表盘，支持参数滑块实时调整（B、n、方法、极端值注入）并即刻看到 CI 变化。两个交互项目均无外部依赖、可直接用于课堂演示和答辩展示。

（6）**小组协作经验**：采用"独立深度执行 + 异步轻协作"模式，三位成员各自负责一个任务的全流程（建模→代码→图表→报告章节），异步群内交流技术问题和解决方案（如中文 CJK 字体方案），最后统一整合。清晰的目录结构（mission1/mission2/mission3）和独立的 README 让互不阻塞，最终统稿时未出现大规模返工。

不足与改进方向：
- 任务一可扩展到更复杂的导热场景（变系数、不规则边界），增加工程真实感；迭代法实现可改用 scipy.sparse 做稀疏矩阵存储，消除 fill-in 问题。
- 任务二的三控开关可进一步用硬件描述语言（如 Verilog）实现，对比软件实现更有工程深度；交互页面可加入动画过渡效果。
- 任务三的 BCa 在中位数下加速度恒为零，若需展示 BCa 的二阶修正优势，应换用光滑统计量（如截尾均值）；覆盖率实验可增加更多分布族（如 log-normal）以更全面评估方法稳健性。
- 三个任务的 matplotlib 配置各自处理，若项目初始化统一写入 `matplotlibrc`，可避免重复踩中文字体坑。
